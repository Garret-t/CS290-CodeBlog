Assignment 4

The most challenging aspect of the assignment was figuring out how to handle different path requests. In Express.js it is relatively simple with app.get('path', callback). However, with just the HTTP module, it was a little more involved. I decided to just make use of the request.url variable and match that to corresponding paths. This may not be the best way to do this, but it gets the job done for this assignment.

From this assignment I learned about the request and response arguments for the HTTP module. I learned how to set the headers and content type of an HTTP transaction. Additionally, I learned about how to actually display data from the server side. Before starting this assignment, I had a vague sense of how HTTP transactions work with GET, POST, PUT, and DELETE requests, but I was not aware of the anatomy with status codes and headers.

The lecture notes were very helpful in completing the assignment. All the necessary functions and variables in the http and fs module were readily available from the lecture notes. Additionally, I looked at the http module documentation to see if there was any easy way to handle routing.

I used google to search for ways to handle routing. I was unnsuccessful in my search as most resources brought up only Express.js. I decided to just go ahead with my routing solution that I mentioned above.

Assignment 3

The most challenging part of this assignment was selecting the proper DOM elements to manipulate. I expected .getElementsByClassName() and .querySelectorAll() to return an array of elements. However, I ran into issues because these .getElementsByClassName() returns an array-like object and .querySelectorAll() returns a NodeList. This caused some issues as it is not as easy to manipulate these data types and iterate over them. Thus, I googled how to convert them to arrays and solved my problem with Array.from().

I learned how to make a page interactive from this assignment. I will definitely be referencing and remembering back to what I have done in this assignment for future web dev endeavors. I already knew some JavaScript prior to this assignment, but this assignment aided in my understanding of the DOM. I thought the way of hiding parts of the page with a class .hidden was neat and also the searching feature seems like it would be helpful in the future.

The lecture notes were the most helpful. Almost all the information I needed was in those notes. I only had to reference the MDN Web Docs a couple times for specifics on what functions returned and what they did. If I ran into an issue, I first checked he lecture notes, if that did not solve my problem, then I checked MDN Web Docs, and as a last resort, I used Google.

As mentioned earlier, I had to Google information on how to use the returned values from .getElementsByClassName() and .querySelectorAll(). Additionally, I Googled the name for the input event, which I could not figure out on my own even though it seems intuitive.
